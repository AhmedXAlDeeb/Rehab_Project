L0
Structure-from-Motion (SfM)
COLMAP · Open3D · ArUco markers · 30-sec video

Patient records 30-sec video of residual limb. Printed ArUco markers provide absolute scale. COLMAP reconstructs sub-2mm accurate 3D mesh.

Projection Model
$\mathbf{x}_i = \mathbf{K}[\mathbf{R}_i \mid \mathbf{t}_i]\,\mathbf{X}$
Bundle Adjustment
$\min \displaystyle\sum_{i,j} \left\|\mathbf{x}_{ij} - \pi(\mathbf{R}_i,\mathbf{t}_i,\mathbf{X}_j)\right\|^2$
ArUco Scale Recovery
$s = L_\text{phys} / L_\text{SfM} \quad \Rightarrow \quad \mathbf{X}_\text{cal} = s\cdot\mathbf{X}_\text{SfM}$

Layer 1
Socket Pressure Simulation
Replacing Physical Test Sockets
L1
MuJoCo Soft-Body Contact Mechanics
Rigid socket · compliant tissue · daily volume fluctuation
Contact Compliance (MuJoCo solref/solimp)
$f_n = f\!\left(d_t, d_r, d_1, d_2\right)$ — skin: $d_t{=}0.02$, socket: $d_t{=}0.001$
Pressure at Contact Point k
$P_k = F_{n,k} / A_k \quad [\text{kPa}] \quad$ threshold: $P_k > 40\,\text{kPa}$ → ulcer risk
Daily Volume Decay (Lilja & Öberg 1997)
$V(t) = V_0 \!\cdot\!\left(1 - \alpha\ln\!\left(1+\tfrac{t}{T_{1/2}}\right)\right), \quad \alpha{\approx}0.05$

The simulator evaluates fit at three time points — morning, midday, evening — to predict how the socket degrades throughout the day as the limb loses volume.

🦴 Rigid Body

Socket mesh: near-rigid. $d_t{=}0.001$, instant response.

🫀 Soft Body

Limb mesh: skin compliance. $d_t{=}0.02$, $d_r{=}1.0$.

📊 Output

2D pressure heatmap per activity (rest, walk, lift 1kg, 5kg).

🔄 Volume

Simulate 3 time points — detect poor evening fit before fabrication.

⚠️ If fit drops below threshold at any time point, design is flagged for a liner recommendation before any physical socket is manufactured.

Layer 3
The Biological Twin
Foundation World Model for sEMG
class BiologicalTwin(nn.Module):
  def __init__(self):
    # Temporal CNN — local activation patterns
    self.cnn = nn.Sequential(
      nn.Conv1d(12, 64,  kernel_size=5),
      nn.BatchNorm1d(64),  nn.ReLU(),
      nn.Conv1d(64, 128, kernel_size=3),
      nn.Conv1d(128,256, kernel_size=3),
      nn.AdaptiveAvgPool1d(32)
    )
    # LSTM — sequential movement context
    self.lstm = nn.LSTM(256, 256,
                  num_layers=2, dropout=0.3)
    # Kinematic decoder → 21 joint angles (°)
    self.decoder = nn.Sequential(
      nn.Linear(256, 128), nn.ReLU(),
      nn.Linear(128, 21)
    )
EWC Loss — Prevents Forgetting
$\mathcal{L}_\text{EWC} = \mathcal{L}_\text{task} + \dfrac{\lambda}{2}\displaystyle\sum_i F_i(\theta_i - \theta^*_i)^2$

Training strategy — four phases, each using a different Ninapro subset:

PhaseDataWhat Updates
Pre-trainDB1+DB2 (67 intact)All layers
Amputee FTDB3+DB10 amputeesAll, low LR
BootstrapL2 session (10 min)LSTM + decoder
Session updatePer-session clean winsLSTM + EWC
Fisher Information (EWC weight)
$F_i = \mathbb{E}\!\left[\!\left(\!\dfrac{\partial \log p(y|\mathbf{x};\theta)}{\partial\theta_i}\!\right)^{\!2}\right]$
ℹ️ Output is continuous 21-DOF kinematics — not 49 discrete classes. Hand moves fluidly, not by snapping between positions.

Layer 4
The Digital Twin Hand
Physics-Based Motor Feedback
🦴 Joint Kinematics

5 fingers × 3 hinge joints = 15 DOF. Anatomically correct ROM, damping, stiffness, coupled tendons.

🤝 Grasp Stability

Contact solver: normal forces + friction cone at all contact points. Grasp stable ↔ friction cone satisfied.

📦 Task Objects

Foam ball (50g), cup (200g), coin (5g), key, elastic band — each stresses a different control aspect.

💪 Fatigue Model

Hold duration limit estimated from session history and updated per session.

Friction Cone — Grasp Success Condition
$\sqrt{f_t^2 + f_b^2} \;\leq\; \mu \cdot f_n \quad$ at all contact points
Reward (extends existing env.py)
$r = 2.0(-d) + 1.2(-\|\Delta a\|^2) + 0.01(-\|a\|^2) + 3.0\cdot\mathbf{1}[\text{stable}]$

Six uses beyond overnight training:

UseWhen
Mirror therapyContinuously, live
Pre-session audit90 sec before session
Task judgeAfter every attempt
Counterfactual replayPost-session review
Electrode optimiserClinic visit
Fatigue pacingContinuously, live
⚠️ Fatigue vs. electrode failure look identical in raw EMG. The digital twin disambiguates: correct response but slow = fatigue. No response + confidence crash = electrode issue.

Layer 5
RL Therapist + Co-Adaptation Loop

A — Curriculum Policy (RL Therapist)

MDP — State / Action / Reward
$\mathcal{S} = [\bar{a}_{1:K}, s_n, f_t, \Delta t_\text{surg}]$
$\mathcal{A} = [\text{task, difficulty, duration}]$
$r = \dot{\bar{a}}_k - \beta_1\mathbf{1}[\bar{a}_k{<}0.4] - \beta_2\mathbf{1}[\bar{a}_k{>}0.95]$

$\dot{\bar{a}}_k$: accuracy slope (learning signal). $\beta_1$: frustration penalty. $\beta_2$: boredom penalty.

B — Overnight Co-Adaptation

Virtual Patient Generator
$\tilde{\mathbf{x}} \sim \mathcal{N}(\boldsymbol{\mu}_c, \boldsymbol{\Sigma}_c) \cdot (1 - \alpha_f f_t)$
PPO Surrogate Objective
$\mathcal{L}^\text{PPO} = \hat{\mathbb{E}}\!\left[\min\!\left(r_t\hat{A}_t,\;\text{clip}(r_t,1{\pm}\varepsilon)\hat{A}_t\right)\right]$
── OVERNIGHT CLOUD LOOP (4h · 100K episodes) ────
Agent 1: Virtual Patient
↓ generates synthetic sEMG(intent, fatigue)
Agent 2: PPO Controller
↓ decodes → commands digital twin → attempts task
MuJoCo physics judges: stable grasp?
↓ PPO updates on reward signal
Agent 1 drifts distribution (next-day drift)
↓ repeat until budget exhausted
Improved weights → patient app before morning
💡 The machine adapts to the patient. The patient wakes up with a better controller — without doing anything.

Task Evaluation
Three-Layer
Attempt Scoring
Composite Score
$S = 0.25\,S_\text{signal} + 0.35\,S_\text{kinematic} + 0.40\,S_\text{physics}$
Layer 1 — Signal Decoding
$S_\text{signal} = p_\text{max} \cdot \mathbf{1}[\hat{c} = c^*]$
Layer 2 — Kinematic Accuracy
$S_\text{kin} = \max\!\!\left(0,1{-}\tfrac{\bar\delta_\theta}{45°}\right) \cdot\min\!\!\left(\tfrac{T_\text{hold}}{2s},1\right)$
Layer 3 — Physics Task
$S_\text{phys} = 0.3\,\mathbf{1}[\text{contact}]+0.3\,\mathbf{1}[\text{stable}]+0.4\,\mathbf{1}[\text{complete}]$
